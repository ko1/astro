# optcarrot 対応: splat 引数 (呼び出し側)

- Date: 2026-04-11
- Commit: 3dfb1ec
- Target: `benchmark/optcarrot/bin/optcarrot-bench` を動かす

## 目的

optcarrot を動かす第一歩として、`nes.rb:15` の
`APU.new(@conf, @cpu, *@audio.spec)` をパースして実行できるようにする。

## 問題の解析

abruby は呼び出し側 splat (`foo(*args)`) をサポートしていなかった。パーサが
Prism::SplatNode に遭遇すると `unsupported node` で停止。

optcarrot 内の splat 使用箇所（初期化・config・cpu・ppu 等）は数カ所で、
特に `nes.rb`, `apu.rb`, `config.rb`, `cpu.rb (send)`, `ppu.rb` にまたがる。
今回はまず call-site での一般的な splat を解消する。

## 解決方針

- 新ノードを 2 つ導入:
  - `node_func_call_apply(name, args_expr, mc)` — 暗黙 self の `foo(*x)`
  - `node_method_call_apply(recv, name, args_expr, mc)` — `obj.m(*x)`
- パーサ: 引数リストに `Prism::SplatNode` が含まれる場合、引数列を
  `[a] + b + [c]` 形式の Array 式に lower し、上記ノードに渡す。
  非 splat 連続部分は `node_ary_new` で、splat 部分は式そのものを使い、
  Array#+ で畳み込む。
- 実行時: `apply_unpack` が Array を評価し `c->fp[arg_index]` に固定
  （GC 対策）。要素を `c->fp[arg_index+1..]` にコピーし、通常の
  `dispatch_method_frame` 経路（cache hit / slow path 両方）にフォール
  バック。静的上限 `ABRUBY_APPLY_MAX_ARGS=32`。
- パーサは call_arg_idx に `1 + 32` スロット予約し、recv / args 子式は
  それ以降のスロットを使う（中間コンパイル時テンポラリが runtime unpack
  領域を踏まないようにする）。

性能最優先ではない（動けばよい）のと、具体化可能な splat はまれなので、
専用の超高速ノードは作らず `+` 畳み込みに一任。

## 実装

- `node.def`: `apply_unpack` ヘルパ + `node_func_call_apply` /
  `node_method_call_apply` を追加
- `abruby.c`: 2 つの alloc wrapper (`rb_alloc_node_*_apply`) と
  `Init_abruby` での登録
- `lib/abruby.rb`: `args_have_splat?`, `build_single_ary`,
  `build_ary_literal`, `build_splat_args_array`, `transduce_splat_call`
  を追加、`transduce_call` の先頭で splat 検知
- ASTroGen はノード定義から DISPATCH / MARK / SPECIALIZE 等を自動生成
  するため、追加の手書き配管は不要

## テスト

- `test/test_splat_call.rb` を新規追加（15 tests）:
  - func_call_apply: splat のみ / 前付き / 後付き / 途中 / 複数 splat /
    変数経由 / 空配列
  - method_call_apply: 明示レシーバ / ネスト call の混在 / 自 self
  - 大きい動的 argc (10 要素)
  - ネスト splat (`f(*g(1,2), *g(3,4))`)
  - 非配列 splat → RuntimeError（`rescue` で捕捉可能）

- 780 → 795 tests, 0 failures, 0 errors（通常ビルド / debug-test 両方）

## ベンチマーク

呼び出しホットパス (`node_func_call` / `node_method_call`) には
変更を加えていないため regression は想定していない。

```
ruby           abruby+compiled    compiled/ruby
dispatch       0.455s             0.26x
fib            0.335s             0.22x
method_call    0.299s             0.19x
```

いずれも既存プロファイルと同等（microbenchmark 幅のゆらぎの範囲内）。

## 既知の制限

- `foo(a, *b) { ... }` のように block を伴う splat call は未サポート
  （`raise` で弾く）。optcarrot では該当パターン未出現。
- 動的 argc > 32 は RuntimeError。
- splat 対象が Array 以外の場合は `to_a` を呼ばない — 直接 raise。
  optcarrot 内では splat 対象は常に Array なので問題なし。
- メソッド定義側 `def f(*args)` は未実装（Phase 2）。
